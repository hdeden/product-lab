# 02_source_notes

## Purpose
- Capture source highlights from HRIS Design v0.1 for downstream synthesis.

## Inputs
- HRIS Design v0.1.pdf

## Outputs
- Consolidated notes on market, needs, features, platform, and roadmap.

## Notes
- Target segment: UMKM cafe and security businesses in Indonesia.
- Core pain points: manual attendance, leave via chat, payroll in spreadsheets; high error risk and time loss.
- Market baseline: common HRIS features include mobile attendance with GPS/selfie, leave approval, payroll automation, and ESS.
- Competitor references: Talenta, Gadjian, Hadirr, GreatDay HR, HRDPintar, Aqtive HR, KantorKu HRIS.
- User needs: simple mobile UI, Bahasa Indonesia support, and easy admin onboarding.
- MVP focus: attendance, leave, payroll basics, and slip gaji digital.
- Differentiation: shift scheduling, multi-location, and field attendance fit for cafe and security workflows.
- Platform: multi-tenant SaaS with mobile apps (Android/iOS) and web admin dashboard.
- Compliance: Indonesian labor rules, BPJS, PPh21; emphasize data privacy and security (statutory payroll in Phase 2).
- Roadmap: MVP -> core feature completeness -> launch/growth -> scale to 10k+ users -> maturity.
