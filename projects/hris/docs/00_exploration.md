# 00_exploration

## Purpose
- Define the initial product scope for an HRMS SaaS targeting Indonesian UMKM in cafe and security services.
- Align on the core problems (manual HR, error-prone payroll, fragmented attendance) and the desired outcomes.

## Inputs
- HRIS Design v0.1 research and plan (projects/hris/HRIS Design v0.1.pdf).
- Market scan of local HRIS/HRMS competitors and common feature baselines.
- Target user context: operational staff, HR admins, managers, business owners.

## Outputs
- Draft scope statement and MVP feature set.
- Target user roles and primary use cases.
- Initial success metrics and adoption goals.
- Assumptions, risks, and open questions.

## Checklist
- [ ] Confirm target segment: UMKM cafe and security companies in Indonesia.
- [ ] Document core pain points: manual attendance, leave by chat, payroll via spreadsheets.
- [ ] Define MVP scope: attendance, leave, payroll basics, and basic ESS.
- [ ] Note platform needs: mobile-first for employees and web dashboard for admins.
- [ ] Capture compliance needs: Indonesian labor rules, BPJS, PPh21, and data privacy.
- [ ] Identify differentiation: sector-specific shift and multi-location support.
